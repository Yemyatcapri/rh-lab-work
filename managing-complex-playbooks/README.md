# RH294 Lab – Managing Complex Plays and Playbooks

## Overview

This lab demonstrates how to organize and execute more complex playbooks using **Ansible**.
The project shows how to structure tasks, manage inventory, and configure Ansible to run automation against managed hosts.

The lab focuses on:

* Structuring playbooks for readability and reuse
* Organizing task files into directories
* Managing configuration with `ansible.cfg`
* Running playbooks using an inventory file
* Testing host connectivity with a separate playbook

## Project Structure

```
managing-complex-playbooks/
│
├── ansible.cfg        # Local Ansible configuration
├── inventory          # Inventory file defining managed hosts
├── playbook.yml       # Main playbook
├── host-test.yml      # Playbook used to test host connectivity
│
├── files/             # Files copied to managed nodes
└── tasks/             # Task files used by the main playbook
```

## Requirements

* Linux system
* Ansible installed
* Access to managed hosts
* SSH connectivity between control node and managed nodes

## Configuration

The `ansible.cfg` file defines local configuration for this project, such as:

* Default inventory location
* SSH settings
* Other runtime options for Ansible

The `inventory` file lists the managed hosts that the playbooks will target.

## Running the Playbooks

### Test host connectivity

```
ansible-navigator run host-test.yml -i inventory
```

### Run the main playbook

```
ansible-navigator run playbook.yml -i inventory
```

## Learning Objectives

This lab demonstrates how to:

* Organize complex Ansible playbooks
* Separate tasks into reusable files
* Use project-level configuration
* Maintain a clean project directory structure

## Notes

Temporary artifact files and logs generated by Ansible Navigator are not included in this repository.
These files are automatically created during playbook execution and are typically excluded using `.gitignore`.
